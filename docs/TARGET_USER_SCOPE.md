# PJT-0001 · TARGET_USER_SCOPE

最后更新：2026-03-07  
状态：**首轮范围已冻结（A01 完成）**

## 1. 结论
本项目首轮不再尝试“同时完整服务所有残障群体”。

### 1.1 主服务人群
**轮椅 / 行动不便用户**

包含但不限于：
- 轮椅用户
- 使用拐杖、助行器的用户
- 老年行动迟缓用户
- 对台阶、长距离步行、复杂换乘明显敏感的人群

### 1.2 次服务人群
**低视力用户**

包含但不限于：
- 需要高对比度、大字号、减少界面噪声的用户
- 依赖更清晰层级、更显著状态提示的用户
- 可结合屏幕放大、基础语音提示使用系统的用户

---

## 2. 首轮承诺支持的能力边界

### 2.1 对主服务人群（轮椅 / 行动不便）
首轮优先建设：
- 更少台阶 / 更少高风险换乘的路线偏好
- 电梯、坡道、站点上下车可达性相关信息的权重优先
- 更清晰的风险提示与“数据不足时降级/拒绝推荐”机制
- 对步行距离、换乘复杂度、关键设施缺失进行更强约束

### 2.2 对次服务人群（低视力）
首轮优先建设：
- 高对比度、大字号、焦点清晰、键盘可达
- 关键状态与错误信息显性化
- 路线结果更可读、可解释、可语音播报
- 降低页面视觉噪声与动态干扰

---

## 3. 当前只做基础可访问支持、暂不承诺完整适配的人群

### 3.1 听障用户
当前阶段提供：
- 文本替代
- 视觉状态提示
- 重要事件不只依赖声音

当前**不承诺**：
- 面向听障用户的专用路线优化模型
- 完整的听障场景实时告警策略闭环

### 3.2 全盲 / 重度视障用户
当前阶段提供：
- 基础语义化结构
- 基础语音播报入口
- 屏幕阅读器基线逐步补齐

当前**不承诺**：
- 全链路逐步式无障碍步行导航
- 基于连续盲道 / 语音过街 / 风险点的可信导航
- 可以独立替代专业无障碍导航工具的能力

### 3.3 其他复杂复合需求人群
当前**不承诺**：
- 多重障碍组合下的个性化路径优化
- 精神/认知障碍场景的专门交互设计
- 全广州范围内稳定一致的可达性结论

---

## 4. 为什么这样冻结范围
1. 当前系统最接近可落地的价值，是帮助**轮椅 / 行动不便用户**规避明显不可达路径与高风险换乘。  
2. 当前前端交互改造最容易直接提升体验的人群，是**低视力用户**。  
3. 对全盲 / 重度视障用户，要做成真正可信的系统，需要更完整的盲道、过街、逐步导航与真实验证数据；当前基础还不够。  
4. 对听障用户，当前更适合先补“信息可见性和文本替代”，而不是宣称已完成专门路线优化。  
5. 毕业设计 / 实验项目更适合先做“小范围但可信”，而不是“大范围但不可靠”。

---

## 5. 对后续开发的直接约束
- 后续路线推荐默认优先服务 `轮椅 / 行动不便` 画像。
- `低视力` 作为次服务人群，优先落在交互和信息表达层。
- `听障` 当前纳入基础可访问性保障，不纳入首轮路线优化主目标。
- `全盲 / 重度视障` 当前不作为“已完成适配”宣传口径，直到具备更完整步行段与真实验证能力。
- 所有文档、README、答辩表述、系统说明，必须使用同一口径。
