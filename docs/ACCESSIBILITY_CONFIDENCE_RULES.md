# PJT-0001 · ACCESSIBILITY_CONFIDENCE_RULES

最后更新：2026-03-07

## 1. 目标
统一本项目中“数据可信度 / 推荐可信度”的口径，避免前端、后端、文档各说各话。

## 2. 置信度分级
| 级别 | 分数区间 | 含义 |
|---|---:|---|
| 高 | `>= 80` | 多个关键字段完整，且命中试点人工样本或结构化字段完整，可作为优先候选 |
| 中 | `>= 60` 且 `< 80` | 结构化字段较完整，但仍需结合风险提示谨慎核对 |
| 低 | `< 60` | 关键字段不足，只能作为辅助参考，必要时应降级或拒绝推荐 |

## 3. 当前路由规划使用的主要因子
- 路线无障碍等级是否存在
- 路线设施说明是否存在
- 语音播报字段是否存在
- 盲道字段是否存在
- 导盲犬字段是否存在
- 电梯/换乘设施字段是否存在
- 路线更新时间是否存在
- 是否命中试点人工样本基础
- 是否命中结构化站点表
- 无障碍综合评分高低

## 4. 决策规则
### 轮椅 / 行动不便
- `轮椅/坡道信息` + `电梯/换乘信息` 同时不足时：`REJECTED`
- 任一关键项不足时：`DEGRADED`

### 低视力
- `语音播报` + `盲道信息` 同时不足时：`REJECTED`
- 任一关键项不足时：`DEGRADED`

### 听障（文本提示优先）
- 文字提示设施不足时：`DEGRADED`
- 当前不作为直接拒绝的主条件

## 5. 当前限制
- 该规则已经统一到代码和治理面板中，但仍属于**项目级规则口径**，不是行业标准。
- 若后续引入真实人工核验和更多外部数据，分值与阈值需要继续迭代。
